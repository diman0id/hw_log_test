Legend
======

Label format:

HW_<category><batch#>{_<subitem#>}

<category>:
    S - Sample from manufacturer.
    B - Bad/buggy device.
    P - Production device.
    C - Competitor product.
    I - Integration partner device.
    R - Reference or development device.
    Other categories may be introduced when needed.

<batch#>:
    A 3-digit number of a group of items which arrives as a single
    batch on a particular date.

<subitem#> (optional, repetitive):
    Should be specified if a particular item consists of several parts
    or a number of similar parts, each of which needs to be identified.
    Such separation may be performed when needed, at a later time.

Example:
    A player sample with an external power supply is received from a
    manufacturer. The player is marked as "HW_S100_1" and the PSU is marked
    as "HW_S100_2". Later a mainboard needs to be umounted, so the mainboard
    is registered as a separate item and labeled "HW_S100_1_1".

Photos:
    All registered hardware items should be photographed. Pictures should
    be stored in "/STORAGE/rushd/HW_LOG". Picture file name should start
    with "HW_..." id, suffixed with "-1", "-2", etc. for multiple pictures.

Log message types:
    - Registered.
    - Taken.
    - Returned.
